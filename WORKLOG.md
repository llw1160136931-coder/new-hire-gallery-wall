### 系统评估与稳定性检查

- 梳理 React + Vite 前端与 Django REST Framework 后端结构。
- 完成前端生产构建、Django 系统检查和后端测试。
- 确认原有作品发布、审核、点赞、投票、课程和个人资料流程可正常运行。
- 识别前端单文件过大、SQLite 并发写入、固定培训日期、固定热门标签、生产安全配置不足等问题。

### 培训期功能

- 新增 `TrainingCamp` 培训期模型。
- 课程、作品、排行榜、上传和投票额度改为按当前培训期隔离。
- 支持配置投稿开始/结束时间、投票开始/结束时间和每人票数。
- 新增当前培训期接口：`GET /api/camps/current/`。
- 前端课程日期和培训期范围改为从后端动态读取，不再写死日期。
- 现有课程和作品通过迁移自动归入默认培训期。

### 作品标签

- 新增真实 `Tag` 标签模型及作品多对多关联。
- 发布和重新提交作品时可填写最多 5 个标签。
- 标签支持自动去除 `#`、清理空格和按大小写去重。
- 新增热门标签接口：`GET /api/tags/popular/`。
- 热门标签只统计当前培训期内已审核发布的作品。
- 作品卡片、详情页和审核预览增加真实标签展示。
- 搜索接口增加按标签搜索能力。
- 为现有演示作品回填“AI 海报”“流程 Demo”“知识地图”等标签。

### 审核工作台运营化

- 将原单条审核列表升级为运营后台式审核工作台。
- 待审列表支持按作品类型、文件类型、学员关键词和提交时间筛选。
- 新增待处理、已选中、风险项、AI/培训作品数量等审核指标。
- 支持全选本页、批量通过和批量打回。
- 打回流程由浏览器弹窗改为面板化表单，并内置常用打回原因模板。
- 审核卡片增加风险提示，包括介绍偏短、缺少作品文件或链接、大文件、外部链接需检查等。
- 审核卡片和预览弹窗支持图片、视频、PDF 与附件链接预览。
- 新增 `WorkReviewLog` 审核日志模型，记录审核作品、审核人、通过/打回动作、原因和时间。
- 新增审核日志接口：`GET /api/works/review-logs/`。
- 新增批量审核接口：`POST /api/works/bulk-review/`。
- 管理员右侧待处理数量支持审核后同步刷新，并增加定时刷新。
- 补充后端测试，覆盖批量审核写入日志和待审列表筛选。
- 验证结果：21 项后端测试通过，前端 Vite 生产构建通过。
- 提交并推送至 GitHub：`92133c6 优化审核工作台`。

### 上传与安全加固

- 上传文件增加真实文件签名检查，防止只伪造 MIME 类型。
- 分片上传完成后生成并保存 SHA-256 摘要。
- 上传会话增加过期时间、个人待处理配额和最大分片限制。
- 同一个 `upload_id` 只能用于一个作品，防止重复消费。
- 新增过期上传清理命令：`python backend/manage.py cleanup_uploads`。
- 生产环境增加强密钥检查、DEBUG 禁用、HTTPS、HSTS、安全 Cookie 和 API 限流配置。
- 生产环境不再由 Django 直接提供媒体文件。

### 并发测试

- 新增可复用压测脚本：`backend/load_tests/run_load_test.py`。
- 测试场景包括登录突发、公共接口、已登录读取和投稿/点赞/投票混合操作。
- SQLite 混合写入测试曾出现约 32% 的 `database is locked` 错误。
- PostgreSQL 初次测试发现默认连接数不足，项目专用实例上限调整为 200。
- 调整后完成 100 个用户同时投稿、点赞和投票测试：共 400 次请求，400 次成功，错误率 0%。
- 确认当前仍需单独优化同一公司网络下的登录和匿名接口限流策略。

## 2026-07-10

### 项目成果概览（汇报版）

本阶段已将“新员工培训作品展示墙”从单纯前端页面推进为具备真实后端、数据库、安全校验、审核流程和互动规则的可落地系统。系统围绕“展示成长、鼓励互动、规范审核、沉淀成果”四个目标，完成了学员端、管理员端和数据层的核心闭环建设。

- 搭建了完整的 React + Django REST Framework 前后端架构，前端负责体验展示，后端统一管理登录、课程、作品、点赞、投票、个人资料和审核数据。
- 接入 JWT 限时 Token 登录机制，用户登录后通过有效期受控的 Token 访问接口，为后续正式部署和权限管理打下基础。
- 建立作品发布、待审核、审核通过、打回修改、重新提交的完整流程，保证学员作品不会绕过管理员直接发布。
- 支持图片、PDF、视频等多种作品材料，并完成大文件切片上传能力，最大支持 500MB，适合培训作品、演示视频和设计稿等真实业务材料。
- 新增多图片作品能力，单个作品最多可上传 10 张图片，满足海报集、界面设计稿、过程截图等展示场景。
- 建立点赞和投票机制，并在后端限制同一用户不能重复点赞同一作品、每人最多 5 票，避免刷票和前端篡改数据。
- 课程表从固定页面升级为真实课程数据展示，支持按日期查看课程，并可以查看课程地点、课程内容、授课师资等详情。
- 完成个人中心模块，支持展示个人资料、头像、毕业院校、性别、星座、MBTI 和个人作品，并提供独立编辑资料页面。
- 建设管理员审核工作台，管理员可以查看待审核作品、预览图片/PDF/视频、通过作品或填写原因打回，审核记录可追溯。
- 将系统数据从前端静态展示迁移到后端数据库，点赞数、投票数、课程时间、作品状态等关键数据均以后端为准，提升数据可信度。
- 增加 PostgreSQL 支持和本地迁移验证，为后续多人使用、并发访问和正式部署做准备。
- 完成并发压测验证，100 个用户同时投稿、点赞和投票场景下 400 次请求全部成功，错误率 0%。
- 完成多轮 UI 优化，参考社区类产品的信息流、个人主页、发布页和审核台设计，使系统更接近可演示、可汇报、可试用的产品形态。
- 建立 GitHub 协作流程，代码已接入版本管理，方便后续与同事共同开发、审查和持续迭代。

### 作品互动与上传能力补强

- 加固作品互动规则：后端限制同一用户对同一作品只能点赞一次、同一作品只能投票一次，每位用户最多 5 票。
- 前端点赞和投票不再直接相信本地计数，统一以后端返回的真实数据刷新作品流和详情页，并补充成功/失败提示。
- 删除发布和重新提交表单中的“图片链接”字段，避免用户困惑，也减少前端可篡改入口。
- 新增多图片作品能力，支持一次上传最多 10 张图片；PDF、视频仍保持单文件切片上传流程。
- 新增 `WorkImage` 图片组模型和 `api.0004_workimage` 迁移，作品接口返回 `images` 列表，管理员后台可查看作品图片组。
- 作品卡片展示首张图片和图片数量，作品详情页展示完整图片组；个人中心重新提交作品时可以替换图片组。

### 安全与数据可信度

- 后端增加重复点赞、重复投票、票数上限、伪造前端状态和课程只读等测试覆盖。
- 课程时间、点赞数、投票数等关键数据以后端为准，前端只负责展示和发起请求，降低通过浏览器直接改数据的风险。
